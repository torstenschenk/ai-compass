## 2) KPI Cards (3-card grid)

### Card 1 — Score
- **Label:** Total Overall Score
- **Value:** `{total_score}/5`
  - Example: **2.6/5**
  - Data source: `responses.total_score`
- **Icon:** ribbon/medal icon
- **Icon container:** rounded square, blue tone
- **aria-label:** `"Total Overall Score"`

### Card 2 — Cluster
- **Label:** Company Cluster
- **Value:** `{cluster_name}`
  - Example: **The Structured Builder**
  - Data source: `responses.cluster_id` joined with `cluster_profiles.cluster_name`
- **Icon:** people/group icon
- **Icon container:** rounded square, purple tone
- **aria-label:** `"Company Cluster"`

### Card 3 — Benchmark
- **Label:** Percentile
- **Value:** `Top {percentile}% in {industry}`
  - Example: **Top 32% in Manufacturing**
  - Data computation:
    - Percentile: compare `responses.total_score` against all responses in DB
    - Industry context: from `companies.industry`
- **Icon:** growth chart icon
- **Icon container:** rounded square, indigo tone
- **aria-label:** `"Percentile Benchmark"`

**Benchmark fallback (MVP)**
- If benchmark is not available yet:
  - Value: **Not available**
  - Optional note (small text): **Benchmark will appear once enough peer data is available.**

## 3) Design & UI Specifications (Header + Cards)

**Layout**
- Centered hero section (title + intro)
- Followed by a 3-column card grid (desktop)
- Stacks into 1 column on mobile

**Typography**
- Titles/scores: bold, high-contrast sans-serif

**Card Aesthetics**
- Background: `#FFFFFF`
- Border radius: `12px–16px`

## 4) Technical Requirements (MVP Implementation)

**Dynamic data injection**
Component must render values from GET `/api/v1/responses/{response_id}?token=...`:

- `total_score` (string, e.g., `"2.6"`) - from `responses.total_score`
- `cluster_name` (string, e.g., `"The Structured Builder"`) - from `cluster_profiles.cluster_name` joined via `responses.cluster_id`
- `percentile` (int/float, e.g., `32`) - computed dynamically by comparing this response's score against all responses
- `industry` (string, e.g., `"Manufacturing"`) - from `companies.industry`

**Formatting rules**
- Score: one decimal, fixed (`2.6/5`)
- Percentile: integer preferred (`Top 32% ...`)

**Accessibility**
- Icons must include descriptive `aria-label`
- Text contrast must satisfy WCAG readability requirements
- Grid must be responsive and keyboard-navigable
